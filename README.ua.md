# Inbound Core — n8n Workflow

🇬🇧 [English](README.md) · 🇸🇰 [Slovenčina](README.sk.md)

---

### Огляд

**Inbound Core** — це автоматизований workflow для n8n, який обробляє вхідні електронні листи, оцінює їх бізнес-релевантність за допомогою AI-моделі (Google Gemini) та — для релевантних повідомлень — автоматично створює документ з комерційною пропозицією, зберігає контактні й цінові дані в Google Sheets і фіксує результат у лозі.

---

### Як це працює

```
Gmail Trigger
     │
     ▼
Google Gemini (AI Intake Assistant)
     │
     ▼
Code in JavaScript1 (парсинг відповіді AI)
     │
     ▼
If — Релевантний?
 ├── НІ  → Надсилання сповіщення адміністратору
 └── ТАК → Створення Google Docs (Пропозиція)
               │
               ▼
           Оновлення документа (заповнення пропозиції)
               │
               ▼
           Code in JavaScript (витяг doc_id)
               │
               ▼
           Додавання / оновлення рядка в Google Sheets (лог)
               │
               ▼
           Оновлення документа (фінальна сума)
```

### Опис вузлів

| Вузол | Тип | Призначення |
|-------|-----|-------------|
| **Gmail Trigger** | Тригер | Перевіряє вхідну пошту кожну хвилину |
| **Message a model** | Google Gemini | Аналізує лист: оцінює релевантність, витягує контактні дані, визначає категорію послуги, отримує ціни |
| **Code in JavaScript1** | Код | Парсить JSON-відповідь AI; позначає нерелевантні листи |
| **If** | Умова | Розгалужує потік: релевантний vs. нерелевантний |
| **Send a message1** | Gmail | Надсилає сповіщення адміністратору, якщо лист не вдалося обробити |
| **Create a document** | Google Docs | Створює новий документ з назвою `Proposal — {Ім'я клієнта}` |
| **Update a document** | Google Docs | Вставляє вміст пропозиції (дані клієнта, послуга, базова ціна) |
| **Code in JavaScript** | Код | Витягує ID щойно створеного документа для подальших оновлень |
| **Append or update row in sheet** | Google Sheets | Записує розрахований результат і деталі обробки (Аркуш 3) |
| **Update a document1** | Google Docs | Додає фінальну суму та дату розрахунку до пропозиції |
| **Google_Sheets_Add_Contact** | AI-інструмент | Зберігає контакт на Аркуші 1 |
| **Google_Sheets_Add_Calculation_Input** | AI-інструмент | Зберігає вхідні дані розрахунку на Аркуші 2 |
| **Google_Sheets_Get_Pricing** | AI-інструмент | Зчитує цінові дані для визначеної категорії послуги |

### Правила релевантності листів

**Релевантний** — відправник:
- Запитує кошторис або консультацію
- Хоче замовити послугу
- Обговорює новий проєкт
- Виявляє інтерес до будь-якої з підтримуваних послуг

**Нерелевантний** — лист є:
- Спамом або маркетинговим розсиланням
- Відгуком на вакансію або резюме
- Особистим повідомленням
- Автоматичним системним сповіщенням

### Вимоги та налаштування

Перед активацією workflow замініть усі плейсхолдери в JSON-файлі:

| Плейсхолдер | Що підставити |
|-------------|---------------|
| `YOUR_CREDENTIAL_ID` | ID облікових даних n8n для кожного сервісу |
| `YOUR_SPREADSHEET_ID` | ID таблиці Google Sheets |
| `YOUR_SPREADSHEET_URL` | Повна URL-адреса таблиці |
| `YOUR_FOLDER_ID` | ID папки Google Drive для пропозицій |
| `YOUR_EMAIL` | Email-адреса адміністратора для сповіщень |
| `YOUR_WEBHOOK_ID` | ID webhook n8n (якщо використовується) |
| `YOUR_INSTANCE_ID` | ID вашого екземпляра n8n |

**Необхідні облікові дані в n8n:**
- Google Gemini (PaLM) API
- Google Sheets OAuth2
- Google Docs OAuth2
- Gmail OAuth2

**Структура Google Sheets:**
- **Аркуш 1 (`Аркуш1`)** — Контакти (`user_id`, `full_name`, `email`, `phone_number`, `status`)
- **Аркуш 2** — Вхідні дані розрахунку (`category_id`, `input_value_1`, `input_value_2`, `raw_email_body`)
- **Аркуш 3 (`Аркуш3`)** — Лог обробки (`calculated_result`, `created_at`, `processing_log`)

### Примітки

- Workflow за замовчуванням **неактивний** — активуйте його вручну після налаштування.
- Gmail Trigger спрацьовує **кожну хвилину**; частоту можна змінити за потреби.
- Якщо відповідь AI-моделі не вдається розпарсити (некоректний JSON), flow завершиться мовчки — рекомендується додати вузол обробки помилок.
- Номери телефонів зберігаються **без** знаку `+` на початку.
- Якщо ім'я в листі не знайдено, контакт зберігається як `Unknown`.
